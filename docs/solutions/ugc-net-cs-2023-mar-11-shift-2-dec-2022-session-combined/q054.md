# Question 54

Consider the following conditional code. which returns a Boolean values if ( (x>25) && (y>100)) return 'false': else if (x≤25) && (v≤100)) return 'true': else if (r> 25) && (v≤100)) return false; else return true; Simplity it by filling in the following blank with a single Boolean expression without changing the behaviour of the conditional code. return true; else return false';

- **1.** x > 25
- **2.** x≤25
- **3.** y > 100
- **4.** y ≤ 100

## Answer

**2. x≤25**

## Detailed solution

Evaluate the four combinations of the two comparisons. The code returns false for (x>25,y>100), true for (x≤25,y≤100), false for (x>25,y≤100), and true for the remaining case (x≤25,y>100). Thus y does not affect the final result: both cases with x≤25 return true and both cases with x>25 return false. The entire conditional is therefore equivalent to returning x≤25.

## Why the other options do not work

Option 1 is the exact opposite of the required result. Options 3 and 4 depend only on y, but the original conditional gives different results for the same y value when x crosses 25.

## Key idea to remember

For a small conditional with Boolean inputs, make a truth table and look for a variable that becomes irrelevant.
