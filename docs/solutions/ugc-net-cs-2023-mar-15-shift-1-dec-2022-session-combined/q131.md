# Question 131

S 0:87081 The incremental model of software development is

- **1.** A reasonable approach when requirements are well defined.
- **2.** A good approach when a working core product is required quickly.
- **3.** The best approach to use for projects with large development teams.
- **4.** A revolutionary model that is not used for commercial products.

## Answer

**2. A good approach when a working core product is required quickly.**

## Detailed solution

The incremental model delivers the system in a sequence of usable releases. A small but functional core can therefore be designed, implemented, and delivered first; later increments add features. This makes it particularly suitable when a working core product is needed quickly.

## Why the other options do not work

Option 1 better characterizes a straightforward plan-driven approach and is not the distinctive advantage asked for. Team size alone does not make the model best, so option 3 is too absolute. Incremental development is widely used commercially, making option 4 false.

## Key idea to remember

Incremental development supplies useful software early and grows it release by release.
