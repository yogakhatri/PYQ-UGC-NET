# Question 11

Recursive functions are executed in a

- **A.** First in first out-order
- **B.** Last in first out-order
- **C.** Parallel fashion
- **D.** Load balancing

## Answer

**B. Last in first out-order**

## Detailed solution

Each recursive call creates a new activation record on the call stack. The most recently created call must finish before its caller can resume, so records are removed in last-in, first-out order.

## Why the other options do not work

FIFO describes a queue, not nested calls. Recursion is not inherently parallel and has nothing to do with load balancing.

## Key idea to remember

Nested procedure calls are managed by a LIFO stack.
