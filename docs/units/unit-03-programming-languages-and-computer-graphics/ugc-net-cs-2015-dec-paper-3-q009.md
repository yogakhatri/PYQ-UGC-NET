# Question 9

*UGC NET CS · 2015 Dec Paper 3 · Language Design and Translation Issues · Static, Dynamic and Object-Oriented Language Classification*

Match the following w.r.t. programming languages : List - I List - II (a) JAVA (i) Dynamically object oriented (b) Python (ii) Statically Non-object oriented (c) Prolog (iii) Statically object oriented (d) ADA (iv) Dynamically non-object oriented Codes : (a) (b) (c) (d)

- **1.** (iii) (i) (ii) (iv)
- **2.** (i) (iii) (ii) (iv)
- **3.** (i) (iii) (iv) (ii)
- **4.** (ii) (iv) (i) (iii)

> [!TIP]
> **Correct answer: No listed option — the standard language classifications do not produce any code shown**

## Solution

Java is statically typed and object-oriented, matching (iii). Python is dynamically typed and object-oriented, matching (i). Prolog is normally treated as dynamically typed and non-object-oriented, matching (iv). Ada is statically typed; modern Ada supports object-oriented programming, also fitting (iii), while an older simplified syllabus may classify it as statically non-object-oriented, (ii). Either way, no offered sequence matches Java=(iii), Python=(i), Prolog=(iv).

## Key Points

- Separate typing time (static/dynamic) from programming paradigm (object-oriented/non-object-oriented).

## Why the other options are incorrect

Option 1 incorrectly makes Prolog statically typed and Ada dynamically typed. Options 2–4 misclassify Java or Python as well. The original item was reported with no valid choice rather than a defensible one-to-one matching.
