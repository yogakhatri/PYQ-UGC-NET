# Question 30

*UGC NET CS · 2015 Dec Paper 3 · Object-Oriented Programming · Polymorphism, Inheritance, Encapsulation and Abstraction*

Match the following with reference to object oriented modelling : List - I List - II (a) Polymorphism (i) Picking both operator and attributes with operations appropriate to model an object (b) Inheritance (ii) Hiding implementation details of methods from users of objects (c) Encapsulation (iii) Using similar operations to do similar things (d) Abstraction (iv) Create new classes from existing class Codes : (a) (b) (c) (d)

- **1.** (iv) (iii) (i) (ii)
- **2.** (iii) (iv) (i) (ii)
- **3.** (iii) (i) (ii) (iv)
- **4.** (iv) (iii) (ii) (i)

> [!TIP]
> **Correct answer: 2. (iii) (iv) (i) (ii)**

## Solution

Polymorphism uses the same or similar operation interface for related behaviors: (a)–(iii). Inheritance creates new classes from existing classes: (b)–(iv). Encapsulation packages an object's attributes with the operations acting on them: (c)–(i). Abstraction hides lower-level method details from users and presents the essential interface: (d)–(ii). The sequence is (iii),(iv),(i),(ii).

## Key Points

- Polymorphism=same interface; inheritance=derive; encapsulation=bundle state/behavior; abstraction=hide detail.

## Why the other options are incorrect

The other codes swap the four core OOP relationships. In particular, inheritance is the existing-to-new-class mechanism, while polymorphism concerns a common operation interface across forms.
