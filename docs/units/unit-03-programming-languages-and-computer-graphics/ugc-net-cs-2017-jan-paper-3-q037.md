# Question 37

*UGC NET CS · 2017 Jan Paper 3 · Object-Oriented Programming · Abstraction versus Encapsulation*

Abstraction and encapsulation are fundamental principles that underlie the object oriented approach to software development. What can you say about the following two statements ? I. Abstraction allows us to focus on what something does without considering the complexities of how it works. II. Encapsulation allows us to consider complex ideas while ignoring irrelevant detail that would confuse us.

- **1.** Neither I nor II is correct.
- **2.** Both I and II are correct.
- **3.** Only II is correct.
- **4.** Only I is correct.

> [!TIP]
> **Correct answer: 4. Only I is correct.**

## Solution

Statement I correctly defines abstraction: expose what an object does while suppressing unnecessary implementation detail. Statement II also describes abstraction—forming a manageable conceptual view by ignoring irrelevant detail—not encapsulation. Encapsulation instead bundles state and behavior inside a class and controls access to the representation. Therefore only I is correctly attributed, giving option 4.

## Key Points

- Abstraction selects the essential view; encapsulation packages implementation and restricts direct access.

## Why the other options are incorrect

Options 2 and 3 accept the mislabeled definition in II. Option 1 rejects the accurate definition of abstraction in I. Information hiding often results from encapsulation, but it is not the same definition as choosing which conceptual details matter.
