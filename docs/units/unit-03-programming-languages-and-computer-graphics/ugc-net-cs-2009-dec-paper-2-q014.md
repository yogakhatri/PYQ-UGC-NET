# Question 14

*UGC NET CS · 2009 Dec Paper 2 · Language Design and Translation Issues · Virtual Computers and Binding Times*

Encapsulation is

- **A.** Dynamic binding
- **B.** A mechanism to associate the code and data.
- **C.** Data abstraction
- **D.** Creating new class

> [!TIP]
> **Correct answer: B. A mechanism to associate the code and data.**

## Solution

Encapsulation bundles an object's data with the methods that operate on that data and controls access through the class interface. It is therefore the mechanism that associates code and data.

## Key Points

- Encapsulation packages state and behavior behind a controlled interface.

## Why the other options are incorrect

Dynamic binding selects implementations at runtime. Abstraction hides unnecessary detail and is supported by encapsulation but is not the bundling definition asked here. Creating a new class is class definition or derivation, not encapsulation itself.
