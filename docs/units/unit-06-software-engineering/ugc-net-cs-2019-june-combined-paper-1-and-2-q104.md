# Question 104

*UGC NET CS · 2019 June Paper 1 And 2 · Software Configuration Management · Software reuse*

Software reuse is:

- **1.** analysing software to recover its design and specification
- **2.** using existing software artifacts and knowledge to build new software
- **3.** reimplementing a legacy system to make it more maintainable
- **4.** analysing software to create a higher-level representation and breaking it into parts to see how it works

> [!TIP]
> **Correct answer: 2. using existing software artifacts and knowledge to build new software**

## Solution

Software reuse means constructing a new system with existing assets such as code, components, frameworks, designs, tests or domain knowledge. It reduces duplicated effort when the reused artifact is suitable and trustworthy.

## Key Points

- Reuse builds new software from existing software artifacts or knowledge; reverse engineering recovers knowledge from existing software.

## Why the other options are incorrect

- **Option 1:** describes reverse engineering.
- **Option 3:** describes re-engineering or modernization.
- **Option 4:** again describes reverse engineering and program analysis.
