# Question 19

*UGC NET CS · 2018 July Paper 2 · Software Quality Assurance · Cleanroom Software Engineering*

Which of the following is not a key strategy followed by the clean room approach to software development ?

- **1.** Formal specification
- **2.** Dynamic verification
- **3.** Incremental development
- **4.** Statistical testing of the system

> [!TIP]
> **Correct answer: 2. Dynamic verification**

## Solution

Cleanroom development combines formal specification, correctness arguments/static verification, incremental development, and statistically designed usage-based testing by an independent certification team. `Dynamic verification` is not one of its key strategies; the development team aims to prevent defects through formal reasoning rather than relying on executing unit tests. Therefore option 2 is correct.

## Key Points

- Cleanroom emphasizes defect prevention by formal specification and static correctness verification, followed by independent statistical certification.

## Why the other options are incorrect

Formal specification defines behavior precisely, incremental development builds certified increments, and statistical testing estimates reliability from expected use. Those are all characteristic Cleanroom practices.
