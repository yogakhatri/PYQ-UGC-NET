# Question 107

*UGC NET CS · 2019 June Paper 1 And 2 · Software Process Models · Selecting a process model*

Match software process models with suitable software systems. List I: (a) Waterfall model; (b) Incremental development; (c) Prototyping; (d) RAD. List II: (i) e-business software that starts with basic functionality and then adds advanced features; (ii) an inventory-control system for a supermarket to be developed within three months; (iii) a virtual-reality system for simulating vehicle navigation on a highway; (iv) automation of a school's manual student-record system.

- **1.** a-ii, b-iv, c-i, d-iii
- **2.** a-i, b-iii, c-iv, d-ii
- **3.** a-iii, b-ii, c-iv, d-i
- **4.** a-iv, b-i, c-iii, d-ii

> [!TIP]
> **Correct answer: 4. a-iv, b-i, c-iii, d-ii**

## Solution

School-record automation has stable, well-understood requirements, fitting waterfall (a-iv). E-business software that begins with a core and grows feature by feature fits incremental development (b-i). A novel virtual-reality navigation system has uncertain technical and user requirements, so prototyping is appropriate (c-iii). A three-month inventory system suits rapid application development (d-ii).

## Key Points

- Choose a process model from requirement stability, novelty, delivery urgency and the value of early user feedback.

## Why the other options are incorrect

Options 1–3 mismatch at least one defining situation, particularly the incremental e-business system, experimental VR prototype or time-boxed RAD project.
