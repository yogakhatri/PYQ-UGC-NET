# Question 60

*UGC NET CS · 2015 June Paper 3 · Software Development Methods and Tools · JAD, CASE, Agile, and Component-Based Development*

Match the following for methods of MIS development : List - I List - II (a) Joint Application Design (JAD) (i) Delivers functionality in rapid iteration measured in weeks and needs frequent communication, development, testing and delivery (b) Computer Aided Software Engg (ii) Reusable applications generally with one specific function. It is closely linked with idea of web services and service oriented architecture. (c) Agile development (iii) Tools to automate many tasks of SDLC (d) Component based technology (iv) A group based tool for collecting user requirements and creating system design. Mostly used in analysis and design stages of SDLC Codes : (a) (b) (c) (d)

- **1.** (i) (iii) (ii) (iv)
- **2.** (iv) (iii) (i) (ii)
- **3.** (iii) (iv) (i) (ii)
- **4.** (iii) (i) (iv) (ii)

> [!TIP]
> **Correct answer: 2. (iv) (iii) (i) (ii)**

## Solution

JAD uses facilitated group sessions to collect requirements and shape designs: (a)–(iv). CASE provides tools that automate SDLC tasks: (b)–(iii). Agile delivers working functionality in short, communication-heavy iterations: (c)–(i). Component-based development assembles reusable focused components and relates naturally to services: (d)–(ii). The code is `(iv),(iii),(i),(ii)`.

## Key Points

- JAD collaborates, CASE automates, Agile iterates rapidly, and component technology reuses packaged functionality.

## Why the other options are incorrect

Options 1, 3, and 4 assign at least one method to the wrong defining feature. `CASE → automation` and `Agile → rapid iterations` are the fastest anchors and leave only option 2.
