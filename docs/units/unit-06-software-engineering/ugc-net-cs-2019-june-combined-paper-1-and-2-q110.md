# Question 110

*UGC NET CS · 2019 June Paper 1 And 2 · Software Requirements · Requirements validation*

Software validation mainly checks for inconsistencies between

- **1.** use cases and user requirements
- **2.** implementation and system design blueprints
- **3.** detailed specifications and user requirements
- **4.** functional specifications and use cases

> [!TIP]
> **Correct answer: 3. detailed specifications and user requirements**

## Solution

Validation asks whether the specified product is the product users actually need. It therefore compares the detailed software specifications with user requirements and expectations. A mismatch means the team may faithfully implement the specification yet still build the wrong system.

## Key Points

- Verification asks 'Are we building the product right?'; validation asks 'Are we building the right product?'.

## Why the other options are incorrect

The other pairs are useful consistency checks during modelling or verification, but they do not express the main validation boundary between the specification and user need.
