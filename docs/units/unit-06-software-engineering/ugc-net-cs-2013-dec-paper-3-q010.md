# Question 10

*UGC NET CS · 2013 Dec Paper 3 · Object-Oriented Analysis and Design · UML Diagram Roles*

Match the following with respect to relationship between objects and classes : List – I List – II a. State diagram i. Useful for both abstract modelling and for designing actual program b. Object diagram ii. Describes object classes c. Class diagram iii. Useful for documenting test cases d. Instance diagram iv. Describing the behaviour of a single class of objects. Codes : a b c d

- **A.** iv i ii iii
- **B.** ii iii iv i
- **C.** iii iv ii i
- **D.** ii iv i iii

> [!TIP]
> **Correct answer: A. iv i ii iii**

## Solution

A state diagram describes the behavior of one object class over time (iv). An object diagram is useful for abstract modeling and concrete program design (i). A class diagram describes object classes and their static relationships (ii). An instance diagram can document concrete test-case configurations (iii). The mapping is iv,i,ii,iii.

## Key Points

- UML cue: class diagram is static type structure; object/instance diagram is a concrete snapshot; state diagram is lifecycle behavior.

## Why the other options are incorrect

The other codes confuse static class descriptions with behavioral state models or swap general object models with concrete instance snapshots. Fixing state→iv and class→ii leaves only option A.
