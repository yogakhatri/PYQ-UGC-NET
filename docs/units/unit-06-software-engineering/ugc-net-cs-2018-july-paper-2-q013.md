# Question 13

*UGC NET CS · 2018 July Paper 2 · Software Configuration Management · Source-Code Management Features*

Which one of the following is not typically provided by Source Code Management Software ?

- **1.** Synchronisation
- **2.** Versioning and Revision history
- **3.** Syntax highlighting
- **4.** Project forking

> [!TIP]
> **Correct answer: 3. Syntax highlighting**

## Solution

Source-code management systems primarily coordinate shared changes, record versions and revision history, and support branching/forking workflows. Syntax highlighting is an editor or integrated-development-environment presentation feature, not a core SCM service. Therefore option 3 is correct.

## Key Points

- SCM tracks and coordinates versions; editors analyze and visually present source text.

## Why the other options are incorrect

Synchronization, version history, and project forks/branches all manage parallel development and code evolution, so options 1, 2, and 4 are typical SCM capabilities. A web interface may display highlighted code, but that convenience is not what makes it source control.
