# Question 43

*UGC NET CS · 2015 June Paper 3 · Software Verification and Validation · Verification versus Validation*

Verification :

- **1.** refers to the set of activities that ensure that software correctly implements a specific function
- **2.** gives answer to the question - Are we building the product right ?
- **3.** requires execution of software
- **4.** both (1) and (2)

> [!TIP]
> **Correct answer: 4. both (1) and (2)**

## Solution

Verification asks, `Are we building the product right?` It comprises activities that check whether work products correctly implement specified functions and conform to preceding specifications. Therefore statements 1 and 2 are true, so option 4 is correct.

## Key Points

- Verification checks conformance to specification; validation checks fitness for user purpose.

## Why the other options are incorrect

Verification does not always require software execution: reviews, inspections, walkthroughs, and static analysis are verification techniques. Statement 3 is false. Validation instead asks whether the right product is being built for user needs.
