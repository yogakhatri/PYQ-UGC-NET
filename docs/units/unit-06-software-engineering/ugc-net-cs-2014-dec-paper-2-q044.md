# Question 44

*UGC NET CS · 2014 Dec Paper 2 · Software Metrics · Effective Metric Attributes*

Which one of the following set of attributes should not be encompassed by effective software metrics ?

- **A.** Simple and computable
- **B.** Consistent and objective
- **C.** Consistent in the use of units and dimensions
- **D.** Programming language dependent

> [!TIP]
> **Correct answer: D. Programming language dependent**

## Solution

An effective software metric should be simple to compute, objective, consistent and dimensionally sensible. It should also be useful across projects and technologies where possible. Dependence on a particular programming language weakens comparability and makes the metric reflect language syntax rather than the underlying product or process quality, so that is the undesirable attribute.

## Key Points

- Good metrics are computable, objective, consistent and broadly comparable—not artifacts of one language.

## Why the other options are incorrect

A makes collection practical. B supports repeatability between evaluators. C prevents meaningless comparisons and arithmetic across incompatible units. All three are desirable metric properties.
