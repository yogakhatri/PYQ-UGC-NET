# Question 15

*UGC NET CS · 2018 July Paper 2 · Software Process Improvement · CMM Maturity Levels*

Match the 5 CMM Maturity levels/CMMI staged representations in List- I with their characterizations in List-II : List - I List - II (a) Initial (i) Processes are improved quantitatively and continually. (b) Repeatable (ii) The plan for a project comes from a template for plans. (c) Defined (iii) The plan uses processes that can be measured quantitatively. (d) Managed (iv) There may not exist a plan or it may be abandoned. (e) Optimizing (v) There’s a plan and people stick to it. Code : (a) (b) (c) (d) (e)

- **1.** (iv) (v) (i) (iii) (ii)
- **2.** (i) (ii) (iv) (v) (iii)
- **3.** (v) (iv) (ii) (iii) (i)
- **4.** (iv) (v) (ii) (iii) (i)

> [!TIP]
> **Correct answer: 4. (iv) (v) (ii) (iii) (i)**

## Solution

Initial processes are ad hoc and a plan may be absent or abandoned, so a→iv. Repeatable has basic project management and a plan people follow, so b→v. Defined uses organization-wide standard processes and plan templates, so c→ii. Managed controls processes with quantitative measures, so d→iii. Optimizing focuses on continual quantitative improvement, so e→i. The sequence (iv),(v),(ii),(iii),(i) is option 4.

## Key Points

- CMM levels build discipline cumulatively: Initial, Repeatable, Defined, Managed, Optimizing.

## Why the other options are incorrect

Options 1–3 move the defined and optimizing characteristics to the wrong levels. The maturity progression is: ad hoc → managed/repeatable projects → standardized defined process → quantitative management → continuous optimization.
