# Question 134

*UGC NET CS · 2019 Dec Paper 1 And 2 · Software Process Models · Agile Models and Practices*

Match each agile process model with its characteristic task or artifact. List I: A. Scrum; B. Adaptive Software Development; C. Extreme Programming; D. Feature-Driven Development. List II: I. CRC cards; II. sprint backlog; III. feature template ‘<action> the <result> <by/for/of/to> a(n) <object>’; IV. time-boxed release plan.

- **1.** A-II, B-IV, C-I, D-III
- **2.** A-I, B-III, C-II, D-IV
- **3.** A-II, B-I, C-IV, D-III
- **4.** A-I, B-IV, C-II, D-III

> [!TIP]
> **Correct answer: 1. A-II, B-IV, C-I, D-III**

## Solution

Scrum maintains a sprint backlog containing the work selected and planned for the current sprint, so A-II. Adaptive Software Development organizes iterative work into time-boxed releases, so B-IV. Extreme Programming uses CRC—class, responsibility, collaborator—cards as a lightweight object-oriented design aid, so C-I. Feature-Driven Development expresses a feature using a short client-valued template such as ‘<action> the <result> <by/for/of/to> a(n) <object>,’ so D-III. The resulting mapping A-II, B-IV, C-I, D-III is option 1.

## Key Points

- Recognize agile methods by their artifacts: Scrum=sprint backlog, ASD=time boxes, XP=CRC cards, FDD=feature template.

## Why the other options are incorrect

Options 2–4 move signature artifacts to the wrong methods. Sprint backlog is specifically Scrum vocabulary, CRC cards are associated with XP's simple collaborative design, and the action-result-object wording identifies an FDD feature.
