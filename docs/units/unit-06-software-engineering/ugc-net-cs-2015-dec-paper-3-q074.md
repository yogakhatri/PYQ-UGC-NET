# Question 74

*UGC NET CS · 2015 Dec Paper 3 · Software Requirements · Analysis, Specification, Validation and Function Points*

Which one of the following statements, related to the requirements phase in Software Engineering, is incorrect ?

- **1.** Requirement validation is one of the activities in the requirements phase.
- **2.** Prototyping is one of the methods for requirement analysis.
- **3.** A modelling-oriented approach is one of the methods for specifying functional specifications.
- **4.** Function points are one of the most commonly used size metrics for requirements.

> [!TIP]
> **Correct answer: 3. A modelling-oriented approach is one of the methods for specifying functional specifications.**

## Solution

The paper intends option 3 as incorrect. Standard requirements work includes elicitation/analysis, specification, and validation; prototyping is an analysis/elicitation technique, and function points estimate functional size from requirements. For functional specification, the conventional label is function-oriented or structured modeling (for example, DFD-based functional decomposition), not the vague category 'modelling-oriented approach' used in statement 3.

## Key Points

- Requirements phase: analyze/elicitate → specify/model → validate; function points measure functional size from the resulting requirements.

## Why the other options are incorrect

Requirement validation checks that documented requirements are correct, complete, consistent, feasible, and testable. Prototypes help stakeholders reveal and refine requirements. Function points count user-visible inputs, outputs, inquiries, files, and interfaces and can be estimated before implementation. A caveat is that formal methods literature does use the term model-oriented specification (such as Z or VDM), so the question's terminology is imprecise; option 3 remains the established exam answer.
