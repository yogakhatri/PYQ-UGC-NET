# Question 14

*UGC NET CS · 2021 Nov With Answers · Software Requirements · Unambiguous SRS*

If every requirement stated in the Software Requirement Specification (SRS) has only one interpretation, then SRS is said to be

- **1.** correct
- **2.** consistent
- **3.** unambiguous
- **4.** verifiable

> [!TIP]
> **Correct answer: 3. unambiguous**

## Solution

An SRS is unambiguous when every stated requirement has exactly one interpretation. That definition matches the condition directly, so option 3 is correct.

## Key Points

- One requirement, one meaning = unambiguous.

## Why the other options are incorrect

Correct means the SRS reflects actual stakeholder needs; consistent means its requirements do not conflict; verifiable means an objective check can determine whether each requirement is satisfied. None of those specifically means one interpretation.
