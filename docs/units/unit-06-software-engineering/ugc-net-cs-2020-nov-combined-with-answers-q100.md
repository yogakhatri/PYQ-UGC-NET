# Question 100

*UGC NET CS · 2020 Nov With Answers · Object-Oriented Analysis and Design · Object-Behavioral Model Construction*

To create an object-behavioural model, which steps does the analyst perform? (A) Evaluates all use cases. (B) Builds a state-transition diagram for the system. (C) Reviews the object-behavioural model for accuracy and consistency. (D) Identifies events that do not drive the interaction sequence.

- **1.** (A), (B) and (C) only
- **2.** (A), (B) and (D) only
- **3.** (B), (C) and (D) only
- **4.** (A), (C) and (D) only

> [!TIP]
> **Correct answer: 1. (A), (B) and (C) only**

## Solution

An object-behavioural model is derived by examining the use cases, recognizing the events that drive object interactions, representing states and event-triggered transitions in state diagrams, and reviewing the resulting model for completeness and consistency. Accordingly A, B, and C are valid steps. D says to identify events that do not drive the interaction sequence; the useful step is the opposite—identify the events that do drive it. Hence option 1.

## Key Points

- Use cases reveal driving events; state diagrams model their effects; review checks the behavioural model.

## Why the other options are incorrect

Options 2–4 include D or omit one of the valid evaluation, modeling, and review steps. A state model must be connected to observable events from the scenarios rather than to irrelevant non-driving events.
