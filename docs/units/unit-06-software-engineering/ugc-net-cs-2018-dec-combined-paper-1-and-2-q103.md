# Question 103

*UGC NET CS · 2018 Dec Paper 1 And 2 · Software Maintenance · Corrective, Adaptive and Perfective Maintenance*

Software products need perfective maintenance for which reason?

- **1.** To rectify bugs observed while the system is in use.
- **2.** When customers need the product to run on new platforms.
- **3.** To support new features that users want it to support.
- **4.** To overcome wear and tear caused by repeated use of the software.

> [!TIP]
> **Correct answer: 3. To support new features that users want it to support.**

## Solution

Perfective maintenance improves or extends a working product in response to new user needs, such as adding requested features or improving performance and usability. Therefore supporting new features that users want is perfective maintenance, option 3.

## Key Points

- Corrective fixes faults, adaptive responds to environment changes, and perfective enhances functionality or quality for users.

## Why the other options are incorrect

Fixing observed defects is corrective maintenance (option 1). Moving the product to a new operating environment or platform is adaptive maintenance (option 2). Software does not physically wear out through repeated execution, so option 4 is not a software-maintenance category.
