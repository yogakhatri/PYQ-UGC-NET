> [!TIP]

## Solution

Condition coverage is white-box (A–III); equivalence partitioning is black-box (B–I); volume testing is performance testing (C–IV); alpha testing is a system-level acceptance activity (D–II).

## Key Points

- Condition coverage sees code; equivalence partitions see input behavior.

## Why the other options are incorrect

The other mappings cross structural, functional, performance, and system testing categories.
