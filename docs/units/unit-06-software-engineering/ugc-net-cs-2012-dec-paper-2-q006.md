# Question 6

*UGC NET CS · 2012 Dec Paper 2 · Software Design · Component-Level Design*

Component-level design is concerned with:

- **1.** Flow-oriented analysis
- **2.** Class-based analysis
- **3.** Both of the above
- **4.** None of the above

> [!TIP]
> **Correct answer: 3. Both of the above**

## Solution

Component-level design translates architectural components into implementable internal designs. In a conventional or structured system it can refine flow-oriented analysis models into procedural components. In an object-oriented system it refines class-based analysis and design classes into methods, interfaces and collaborations. Thus component-level design applies to both perspectives.

## Key Points

- Component-level design is the detailed design of implementable units and can be derived from either flow-oriented or class-based models.

## Why the other options are incorrect

Option 1 is incomplete because component design is not limited to conventional flow-oriented models. Option 2 is incomplete because it is not limited to object-oriented classes. Since both are valid contexts, 'none' is false.
