# Question 44

*UGC NET CS · 2009 Dec Paper 2 · Software Testing · Basis-Path and Control-Structure Testing*

Black Box testing is done

- **A.** to show that s/w is operational at its interfaces i.e . input and output.
- **B.** to examine internal details of code.
- **C.** at client side.
- **D.** none of above.

> [!TIP]
> **Correct answer: A. to show that s/w is operational at its interfaces i.e . input and output.**

## Solution

Black-box testing treats the implementation as hidden and checks observable behavior at its interfaces: supplied inputs, produced outputs, and conformance to requirements. Option A states this purpose.

## Key Points

- Black box tests external behavior without using internal implementation knowledge.

## Why the other options are incorrect

Examining internal code structure is white-box testing. Black-box testing is not defined by being conducted only at a client's site.
