# Question 71

*UGC NET CS · 2023 Mar 15 Shift 1 Dec 2022 Session · Software Design · Good coding practices*

Which of the following is not one of the principles of good coding?

- **1.** Create unit tests before you begin coding.
- **2.** Create visual layout that aids understanding.
- **3.** Refractor the code after you completes the first coding pass.
- **4.** Write self-documenting code not program documentation.

> [!TIP]
> **Correct answer: 4. Write self-documenting code not program documentation.**

## Solution

Unit tests before implementation (test-first development), readable visual layout, and refactoring after an initial pass are recognized coding practices. Self-documenting code reduces comment burden but does not eliminate necessary program/API/design documentation, so statement 4 is not a sound principle as worded.

## Key Points

- Clear code complements documentation; it does not replace all documentation.

## Why the other options are incorrect

Options 1–3 improve correctness, readability, or maintainability.
