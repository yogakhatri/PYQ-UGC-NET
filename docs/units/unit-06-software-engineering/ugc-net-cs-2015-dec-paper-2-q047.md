# Question 47

*UGC NET CS · 2015 Dec Paper 2 · Software Process Models · Prototyping, Iterative and Timeboxed Models*

Which of the following is not a software process model ?

- **1.** Prototyping
- **2.** Iterative
- **3.** Timeboxing
- **4.** Glassboxing

> [!TIP]
> **Correct answer: 4. Glassboxing**

## Solution

Prototyping and iterative development are established software process approaches. Timeboxing is also a process/scheduling approach in which each iteration receives a fixed time window and delivers the highest-priority achievable scope. ‘Glassboxing’ is not a software process model; glass-box/white-box terminology belongs to testing based on internal structure.

## Key Points

- Process models organize development; glass-box describes structural testing, not a development lifecycle.

## Why the other options are incorrect

Options 1–3 all describe recognized ways of organizing development. Choosing glassboxing avoids confusing a test-design perspective with a lifecycle process model.
