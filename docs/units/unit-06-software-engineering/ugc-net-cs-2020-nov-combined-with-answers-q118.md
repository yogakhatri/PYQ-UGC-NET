# Question 118

*UGC NET CS · 2020 Nov With Answers · Software Process Improvement · CMM Maturity Levels*

With reference to the Software Engineering Institute’s Capability Maturity Model (CMM), match (A) Initial, (B) Repeatable, (C) Defined, (D) Managed with (I) process measurement, (II) process definition, (III) project management, (IV) ad hoc.

- **1.** A-III, B-IV, C-II, D-I
- **2.** A-IV, B-III, C-I, D-II
- **3.** A-IV, B-III, C-II, D-I
- **4.** A-III, B-IV, C-I, D-II

> [!TIP]
> **Correct answer: 3. A-IV, B-III, C-II, D-I**

## Solution

At CMM level 1, Initial, work is unpredictable and ad hoc: A→IV. Level 2, Repeatable, establishes basic project-management discipline so successful practices can be repeated: B→III. Level 3, Defined, standardizes and documents organization-wide process definitions: C→II. Level 4, Managed, uses quantitative process measurement and control: D→I. Thus option 3.

## Key Points

- CMM ladder: Initial/ad hoc, Repeatable/project management, Defined/process definition, Managed/measurement, Optimizing/improvement.

## Why the other options are incorrect

The options mainly confuse adjacent maturity levels. The progression is ad hoc → managed projects → defined organizational process → quantitatively measured process; level 5 would add continuous optimization.
