# Question 12

*UGC NET CS · 2018 July Paper 2 · Software Engineering Foundations · Requirements, Design, Validation and Cost Techniques*

Match the following in Software Engineering : List - I List - II (a) Product Complexity (i) Software Requirements Definition (b) Structured System Analysis (ii) Software Design (c) Coupling and Cohesion (iii) Validation Technique (d) Symbolic Execution (iv) Software Cost Estimation Code : (a) (b) (c) (d)

- **1.** (ii) (iii) (iv) (i)
- **2.** (iii) (i) (iv) (ii)
- **3.** (iv) (i) (ii) (iii)
- **4.** (iii) (iv) (i) (ii)

> [!TIP]
> **Correct answer: 3. (iv) (i) (ii) (iii)**

## Solution

Product complexity is a product cost driver in software cost estimation, so a→iv. Structured system analysis models and defines what the system must do, so b→i (requirements definition). Coupling and cohesion are central software-design quality concepts, so c→ii. Symbolic execution explores program behavior using symbolic inputs and is a validation/testing technique, so d→iii. The sequence (iv),(i),(ii),(iii) is option 3.

## Key Points

- Cost drivers estimate effort; structured analysis supports requirements; coupling/cohesion guide design; symbolic execution supports validation.

## Why the other options are incorrect

The other codes place at least one concept in the wrong lifecycle activity. A useful anchor is that coupling/cohesion evaluate modular design, while symbolic execution reasons about execution paths to find or rule out defects.
