# Question 69

*UGC NET CS · 2020 Nov With Answers · Object-Oriented Analysis and Design · UML Static and Behavioral Views*

Which of the following UML diagrams has a static view?

- **1.** Collaboration diagram
- **2.** Use-Case diagram
- **3.** State chart diagram
- **4.** Activity diagram

> [!TIP]
> **Correct answer: 2. Use-Case diagram**

## Solution

Among the listed choices, a use-case diagram gives the static use-case view: it records actors, use cases, and their relationships without ordering interactions in time. A collaboration diagram is an interaction view showing message exchange among objects. State-chart and activity diagrams both model changing behaviour—state transitions and workflow/control flow, respectively. Hence the static-view choice is the use-case diagram, option 2.

## Key Points

- Static diagrams show enduring elements and relations; dynamic diagrams show messages, transitions, or activity flow.

## Why the other options are incorrect

Collaboration, state-chart, and activity diagrams all emphasize runtime behaviour or interaction. Although use cases describe system behaviour at a requirements level, the diagram itself is a static structural snapshot of actors and use-case relationships in this classification.
