# Question 84

*UGC NET CS · 2019 Dec Paper 1 And 2 · Software Project Management · Activity Networks and Critical Path*

Fill the blanks: An ___ chart represents a project schedule as a directed graph. The critical path is the ___ sequence of ___ tasks, and it defines project ___.

- **1.** Activity. Shortest, Independent, Cost
- **2.** Activity. Longest. Dependent. Duration
- **3.** Activity. Longest, Independent. Duration
- **4.** Activity. Shortest, Dependent. Duration

> [!TIP]
> **Correct answer: 2. Activity. Longest. Dependent. Duration**

## Solution

An activity chart or activity network represents project activities and their precedence relationships as a directed graph. A path represents a dependent sequence of tasks. The critical path is the path with the longest total duration; since those tasks have no scheduling slack, that total determines the minimum project duration. The blanks are Activity, Longest, Dependent, Duration—option 2.

## Key Points

- Critical path = longest-duration precedence path; delay on it delays the project.

## Why the other options are incorrect

The shortest path does not control project completion, eliminating options 1 and 4. Tasks on one path are linked by precedence and are therefore dependent rather than independent, eliminating option 3. Critical-path analysis determines time, not directly project cost.
