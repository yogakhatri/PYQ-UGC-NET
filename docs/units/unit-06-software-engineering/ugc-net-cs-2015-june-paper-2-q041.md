# Question 41

*UGC NET CS · 2015 June Paper 2 · Software Metrics · Size- and Function-Oriented Metrics*

Match the following: (a) Size-oriented metrics (b) Function-oriented metrics (c) Extended Function Point metrics (d) Function point; with (i) uses the number of external interfaces as a measurement parameter (ii) was originally designed for business information systems (iii) normalizes quality and/or productivity measures by software size (iv) uses algorithm characteristics as a measurement parameter. Codes: (a) (b) (c) (d)

- **1.** (iii) (iv) (i) (ii)
- **2.** (ii) (i) (iv) (iii)
- **3.** (iv) (ii) (iii) (i)
- **4.** (iii) (i) (iv) (ii)

> [!TIP]
> **Correct answer: 4. (iii) (i) (iv) (ii)**

## Solution

Size-oriented metrics normalize quality or productivity by software size, such as defects per KLOC: (a)–(iii). Function-oriented metrics use functional counts that include external interfaces: (b)–(i). Extended Function Point metrics add algorithmic complexity so they apply beyond data-dominated systems: (c)–(iv). The original function-point method was designed primarily for business information systems: (d)–(ii). The code is `(iii), (i), (iv), (ii)`.

## Key Points

- Size metrics normalize by KLOC; function metrics count external functionality; extended FP also accounts for algorithmic complexity.

## Why the other options are incorrect

Options 1–3 each swap at least one defining characteristic. The easiest anchors are size normalization for size-oriented metrics and algorithm characteristics for the extended function-point approach; these immediately lead to option 4.
