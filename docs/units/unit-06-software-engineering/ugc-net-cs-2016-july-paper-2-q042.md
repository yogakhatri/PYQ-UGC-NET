# Question 42

*UGC NET CS · 2016 July Paper 2 · Software Process Models · Evolutionary Development for Unclear Requirements*

The ________ model is preferred for software development when the requirements are not clear.

- **1.** Rapid Application Development
- **2.** Rational Unified Process
- **3.** Evolutionary Model
- **4.** Waterfall Model

> [!TIP]
> **Correct answer: 3. Evolutionary Model**

## Solution

When requirements are unclear, an evolutionary model develops an initial version and refines it through repeated stakeholder feedback. Requirements can become clearer as users evaluate working increments or prototypes. Therefore option 3 is the best fit.

## Key Points

- Evolutionary development learns and refines requirements through successive working versions and feedback.

## Why the other options are incorrect

Waterfall expects requirements to be substantially understood before sequential development. RAD also needs a reasonably modular, well-scoped system and rapid decisions. RUP is iterative, but the generic textbook model directly associated with initially unclear and changing requirements is the evolutionary model.
