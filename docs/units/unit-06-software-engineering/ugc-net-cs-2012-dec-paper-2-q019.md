# Question 19

*UGC NET CS · 2012 Dec Paper 2 · Software Quality and Process Improvement · CMM Maturity Levels*

The maturity levels used to measure a process are:

- **1.** Initial, Repeatable, Defined, Managed, Optimized
- **2.** Primary, Secondary, Defined, Managed, Optimized
- **3.** Initial, Stating, Defined, Managed, Optimized
- **4.** None of the above

> [!TIP]
> **Correct answer: 1. Initial, Repeatable, Defined, Managed, Optimized**

## Solution

The original Software Capability Maturity Model describes five ordered process-maturity levels: 1 Initial, 2 Repeatable, 3 Defined, 4 Managed, and 5 Optimizing (often written 'Optimized' in exam options). They progress from ad hoc work to disciplined, measured and continuously improving processes.

## Key Points

- Memorize the classic CMM sequence as I-R-D-M-O: Initial, Repeatable, Defined, Managed, Optimizing.

## Why the other options are incorrect

'Primary' and 'Secondary' are not CMM maturity levels, and neither is 'Stating'. Option 1 is the only list that preserves the recognized five-level sequence, so 'none' is false.
