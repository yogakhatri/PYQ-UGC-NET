# Question 101

*UGC NET CS · 2020 Nov With Answers · Software Testing · Black-Box Behavioral Testing*

Which are behavioural (black-box) testing techniques? (A) Equivalence partitioning, (B) graph-based testing method, (C) boundary-value analysis, (D) data-flow testing, (E) loop testing.

- **1.** (B) and (D) only
- **2.** (A), (B) and (C) only
- **3.** (D) and (E) only
- **4.** (A), (C) and (E) only

> [!TIP]
> **Correct answer: 2. (A), (B) and (C) only**

## Solution

Behavioural or black-box testing derives tests from externally visible requirements rather than from program structure. Equivalence partitioning groups inputs expected to behave alike; boundary-value analysis targets class edges; and graph-based testing models externally visible objects, states, or relationships. Thus A, B, and C are behavioural techniques. Data-flow testing follows definition–use paths and loop testing examines loop structure, so D and E are white-box techniques. Therefore option 2.

## Key Points

- Black-box techniques use the specification; data-flow and loop testing use internal code structure.

## Why the other options are incorrect

Option 1 mixes a black-box graph method with white-box data-flow testing. Option 3 contains two structural techniques. Option 4 incorrectly includes loop testing and omits graph-based testing.
