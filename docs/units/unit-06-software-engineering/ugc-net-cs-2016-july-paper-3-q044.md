# Question 44

*UGC NET CS · 2016 July Paper 3 · Software Maintenance · Corrective, Adaptive, Perfective and Preventive Maintenance*

Match the software maintenance activities in List – I to its meaning in List – II. List – I List – II I. Corrective (a) Concerned with performing activities to reduce the software complexity thereby improving program understandability and increasing software maintainability. II. Adaptive (b) Concerned with fixing errors that are observed when the software is in use. III. Perfective (c) Concerned with the change in the software that takes place to make the software adaptable to new environment (both hardware and software). IV. Preventive (d) Concerned with the change in the software that takes place to make the software adaptable to changing user requirements. Codes : I II III IV

- **1.** (b) (d) (c) (a)
- **2.** (b) (c) (d) (a)
- **3.** (c) (b) (d) (a)
- **4.** (a) (d) (b) (c)

> [!TIP]
> **Correct answer: 2. (b) (c) (d) (a)**

## Solution

Corrective maintenance fixes observed errors, so I→(b). Adaptive maintenance accommodates a changed hardware/software environment, so II→(c). Perfective maintenance changes functionality or performance for evolving user needs, so III→(d). Preventive maintenance restructures code to reduce complexity and future maintenance risk, so IV→(a). This is option 2.

## Key Points

- Corrective=bugs; adaptive=environment; perfective=user enhancements; preventive=future maintainability/reliability.

## Why the other options are incorrect

The other codes confuse environment adaptation, requirement enhancement, defect repair, and maintainability improvement. Identify the trigger: bug, environment, user improvement, or future-risk reduction.
