# Question 99

*UGC NET CS · 2020 Nov With Answers · Software Quality and Reliability · Reliability Time Domains*

Software reliability may be described with respect to which time domains? (A) Execution time, (B) calendar time, (C) clock time.

- **1.** (A) and (B) only
- **2.** (B) and (C) only
- **3.** (A), (B) and (C)
- **4.** (A) and (C) only

> [!TIP]
> **Correct answer: 3. (A), (B) and (C)**

## Solution

Software-reliability models can use three common time domains. Execution time is the CPU time actually spent running the software. Calendar time is ordinary elapsed date-and-time over development or operation. Clock time is elapsed running-system time from start to finish, including time the program waits while the computer remains in operation. Therefore reliability can be described with respect to A, B, and C, option 3.

## Key Points

- Reliability has multiple possible time bases: CPU execution time, ordinary calendar time, and elapsed clock time.

## Why the other options are incorrect

Options 1, 2, and 4 each omit one recognized time domain. A model may prefer execution time because it more closely represents operational stress, but that preference does not make calendar or clock-time descriptions invalid.
