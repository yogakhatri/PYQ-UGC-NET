# Question 2

*UGC NET CS · 2013 June Paper 3 · Software Metrics and Models · Cost, Maintenance, Planning and Reliability Models*

Match the models with their uses: a. Walston-Felix model; b. Quick-Fix model; c. Putnam resource-allocation model; d. Logarithmic-Poisson model. i. Failure intensity; ii. Cost estimation; iii. Project planning; iv. Maintenance.

- **A.** ii, i, iv, iii
- **B.** i, ii, iv, iii
- **C.** ii, i, iii, iv
- **D.** ii, iv, iii, i

> [!TIP]
> **Correct answer: D. ii, iv, iii, i**

## Solution

Walston-Felix is an empirical software cost/effort-estimation model, so a→ii. The Quick-Fix model describes an ad hoc maintenance approach, so b→iv. Putnam's resource-allocation model uses the Rayleigh staffing curve and software equation for project planning, so c→iii. The Musa-Okumoto logarithmic-Poisson reliability model describes changing failure intensity during testing, so d→i. The sequence is ii, iv, iii, i.

## Key Points

- Model cues: Walston-Felix=cost, Quick-Fix=maintenance, Putnam=resource/project planning, logarithmic-Poisson=reliability intensity.

## Why the other options are incorrect

A and C incorrectly associate Quick-Fix with failure intensity, while B moves Walston-Felix away from cost estimation. Checking the distinctive endpoints-cost estimation for Walston-Felix and failure intensity for logarithmic-Poisson-leaves only D.
