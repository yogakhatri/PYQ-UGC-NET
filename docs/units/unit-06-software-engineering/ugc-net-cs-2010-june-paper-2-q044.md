# Question 44

*UGC NET CS · 2010 June Paper 2 · Software Design · UI and Component-Level Design*

Design phase will usually be

- **A.** top-down
- **B.** bottom-up
- **C.** random
- **D.** centre fringing

> [!TIP]
> **Correct answer: A. top-down**

## Solution

Software design usually begins with the overall architecture and successively decomposes it into subsystems, modules, interfaces, and detailed operations. This is a top-down progression.

## Key Points

- Move from system architecture to increasingly detailed component design.

## Why the other options are incorrect

Bottom-up design can be used when composing reusable components but is not the usual answer to the general lifecycle question. Random and 'centre fringing' are not disciplined design strategies.
