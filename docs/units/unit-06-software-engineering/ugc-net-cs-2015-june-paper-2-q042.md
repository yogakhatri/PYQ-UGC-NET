# Question 42

*UGC NET CS · 2015 June Paper 2 · Software Testing · Validation Testing*

In which testing strategy requirements established during requirements analysis are validated against developed software ?

- **1.** Validation testing
- **2.** Integration testing
- **3.** Regression testing
- **4.** System testing

> [!TIP]
> **Correct answer: 1. Validation testing**

## Solution

Validation testing checks the completed software against the requirements established during requirements analysis: it asks whether the right product was built for its intended use. Because the question explicitly describes validating those requirements against the developed software, option 1 is direct.

## Key Points

- Validation compares the developed product with user and requirements-analysis expectations.

## Why the other options are incorrect

Integration testing focuses on interfaces among combined components. Regression testing checks that changes did not break previously working behavior. System testing evaluates the complete integrated system in its environment, but the named requirement-conformance strategy in the wording is validation testing.
