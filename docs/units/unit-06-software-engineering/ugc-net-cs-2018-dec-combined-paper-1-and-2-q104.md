# Question 104

*UGC NET CS · 2018 Dec Paper 1 And 2 · Object-Oriented Analysis and Design · UML Diagram Roles*

Match each UML diagram in List I with its description in List II. List I: (a) State diagram, (b) Use-case diagram, (c) Class diagram, (d) Activity diagram. List II: (i) describes how external entities such as people or devices interact with the system; (ii) describes the static or structural view of a system; (iii) shows the flow of a business process, the steps of a use case, or the logic of an object's behaviour; (iv) describes the dynamic behaviour of objects and may describe the entire system behaviour.

- **1.** (a)-(i), (b)-(iv), (c)-(ii), (d)-(iii)
- **2.** (a)-(iv), (b)-(ii), (c)-(i), (d)-(iii)
- **3.** (a)-(i), (b)-(iv), (c)-(iii), (d)-(ii)
- **4.** (a)-(iv), (b)-(i), (c)-(ii), (d)-(iii)

> [!TIP]
> **Correct answer: 4. (a)-(iv), (b)-(i), (c)-(ii), (d)-(iii)**

## Solution

A state diagram models the dynamic states and transitions of an object or system, so (a)→(iv). A use-case diagram shows actors/external entities interacting with the system, so (b)→(i). A class diagram gives the static structural view of classes and relationships, so (c)→(ii). An activity diagram models workflow, business-process steps, or behavioural logic, so (d)→(iii). The complete mapping is (a)-(iv), (b)-(i), (c)-(ii), (d)-(iii), option 4.

## Key Points

- Remember UML roles by four keywords: state—transitions, use case—actors, class—structure, activity—flow.

## Why the other options are incorrect

Options 1–3 exchange the characteristic roles of state, use-case, class, or activity diagrams. In particular, class diagrams are structural, use-case diagrams are actor-facing, activity diagrams show flow, and state diagrams show state-dependent dynamics.
