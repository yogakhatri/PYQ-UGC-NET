# Question 7

*UGC NET CS · 2013 Dec Paper 3 · Software Configuration Management · Change and Version Control*

A process which defines a series of tasks that have the following four primary objectives is known as 1. to identify all items that collectively define the software configuration. 2. to manage chan ges to one or more of these items. 3. to facilitate the construction of different versions of an application. 4. to ensure that software quality is maintained as the configuration evolves over time.

- **A.** Software Qu ality Management Process
- **B.** Software Configuration Management Process
- **C.** Software Version Management Process
- **D.** Software Change Management Process

> [!TIP]
> **Correct answer: B. Software Configuration Management Process**

## Solution

Software configuration management identifies configuration items, establishes baselines, controls and records changes, constructs and tracks versions, and audits evolving configurations for integrity and quality. Those are exactly the four objectives described.

## Key Points

- SCM unifies configuration identification, version control, change control, status accounting and audits.

## Why the other options are incorrect

Quality, version and change management each cover part of the list, but SCM is the umbrella discipline integrating identification, change control, version construction and configuration auditing.
