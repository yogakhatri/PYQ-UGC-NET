# Question 117

*UGC NET CS · 2020 Nov With Answers · Entity–Relationship Modeling · ER Diagram Symbols*

Match the E-R symbols in List I with their descriptions in List II. List I: A. double rectangle; B. oval with the attribute name underlined; C. double oval; D. an entity connected to a relationship by a double line. List II: I. Key attribute type; II. Weak entity type; III. Total participation of entity in a relation; IV. Multivalue attribute type.

- **1.** A-II, B-IV, C-III, D-I
- **2.** A-IV, B-I, C-II, D-III
- **3.** A-II, B-I, C-IV, D-III
- **4.** A-III, B-IV, C-I, D-II

> [!TIP]
> **Correct answer: 3. A-II, B-I, C-IV, D-III**

## Solution

In Chen ER notation, a double rectangle denotes a weak entity, so A→II. Underlining an attribute name marks a key attribute, so B→I. A double oval denotes a multivalued attribute, so C→IV. A double line from an entity to a relationship means total participation, so D→III. The complete mapping is option 3.

## Key Points

- ER visual mnemonic: double rectangle=weak entity, underline=key, double oval=multivalued, double line=total participation.

## Why the other options are incorrect

The distractors swap the visual conventions. A weak entity is not a multivalued attribute, and total participation is shown by a double connecting line rather than by an oval or rectangle alone.
