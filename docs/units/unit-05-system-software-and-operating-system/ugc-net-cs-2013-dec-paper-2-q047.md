# Question 47

*UGC NET CS · 2013 Dec Paper 2 · Memory Management · Set-Associative TLB Tags*

The virtual address generated by a CPU is 32 bits. The Translation Look-aside Buffer (TLB) can hold total 64 page table entries and a 4-way set associative (i.e. with 4- cache lines in the set). The page size is 4 KB. The mini mum size of TLB tag is

- **A.** 12 bits
- **B.** 15 bits
- **C.** 16 bits
- **D.** 20 bits

> [!TIP]
> **Correct answer: C. 16 bits**

## Solution

A 4 KB page uses 12 offset bits, leaving a 20-bit virtual page number in a 32-bit address. A 64-entry, 4-way TLB has 64/4=16 sets, requiring 4 set-index bits. The remaining virtual-page-number bits form the tag: 20-4=16 bits.

## Key Points

- Set-associative TLB tag bits = virtual-address bits - page-offset bits - set-index bits.

## Why the other options are incorrect

12 bits is the page offset, not the tag. 20 bits would be the tag only for a fully associative TLB with no set index. 15 bits does not follow from either the page size or 16-set organization.
