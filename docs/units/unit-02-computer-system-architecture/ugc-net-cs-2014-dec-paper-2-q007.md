# Question 7

*UGC NET CS · 2014 Dec Paper 2 · Digital Codes · Excess-3 Self-Complementing Code*

The Excess-3 decimal code is a self-complementing code because

- **A.** The binary sum of a code and its 9’s complement is equal to 9.
- **B.** It is a weighted code.
- **C.** Complement can be generated by inverting each bit pattern.
- **D.** The binary sum of a code and its 10’s complement is equal to 9.

> [!TIP]
> **Correct answer: C. Complement can be generated by inverting each bit pattern.**

## Solution

Excess-3 encodes decimal digit d as the four-bit binary value d+3. Bitwise complementing that code produces 15−(d+3)=12−d=(9−d)+3, which is precisely the Excess-3 code for the decimal 9's complement 9−d. Therefore the complement digit is generated simply by inverting every bit.

## Key Points

- In Excess-3, one's-complementing all four code bits maps digit d directly to digit 9−d.

## Why the other options are incorrect

Excess-3 is nonweighted, so B is false. A and D describe numerical sums rather than the defining bitwise self-complement property; D also invokes the wrong decimal complement.
