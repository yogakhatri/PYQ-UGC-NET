# Question 89

*UGC NET CS · 2018 Dec Paper 1 And 2 · Syntax Analysis · LL(1) and LR(1) Grammar Tests*

For grammar S->A|B, A->a|c, B->b|c, consider S1: LR(1) can parse all strings generated by G. S2: LL(1) can parse all strings generated by G. Which statements are correct?

- **1.** Only S1
- **2.** Only S2
- **3.** Both S1 and S2
- **4.** Neither S1 nor S2

> [!TIP]
> **Correct answer: 4. Neither S1 nor S2**

## Solution

The string c has two parse trees: S⇒A⇒c and S⇒B⇒c. Thus G is ambiguous. Every LR(1) grammar is unambiguous, so S₁ is false. For LL(1), the alternatives S→A and S→B must have disjoint FIRST sets, but FIRST(A)={a,c} and FIRST(B)={b,c} overlap on c. Therefore S₂ is also false. Neither statement is correct, option 4.

## Key Points

- One string with two derivations proves ambiguity; overlapping FIRST sets of alternatives disprove LL(1).

## Why the other options are incorrect

Options 1–3 accept LR(1), LL(1), or both despite the explicit conflict on terminal c. A parser might recognize the language {a,b,c} using another grammar, but these statements concern the given grammar G.
