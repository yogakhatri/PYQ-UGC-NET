# Question 36

*UGC NET CS · 2013 June Paper 3 · Context-Free Grammars · Dyck Language and Balanced Prefixes*

The grammar with production rules S → aSb |SS|λ generates language L given by :

- **A.** L = {w ∈{a, b}* | na(w) = nb(w) and n a(v) ≥ n b(v) where v is any prefix of w}
- **B.** L = {w ∈{a, b}* | na(w) = nb(w) and n a(v) ≤ n b(v) where v is any prefix of w}
- **C.** L = {w ∈{a, b}* | n a(w) ≠ nb(w) and na(v) ≥ nb(v) where v is any prefix of w}
- **D.** L = {w ∈{a, b}* | na(w) ≠ nb(w) and n a(v) ≤ n b(v) where v is any prefix of w}

> [!TIP]
> **Correct answer: A. L = {w ∈{a, b}* | na(w) = nb(w) and n a(v) ≥ n b(v) where v is any prefix of w}**

## Solution

Treat a as an opening bracket and b as a closing bracket. The rule aSb wraps a balanced string in one matching pair, SS concatenates two balanced strings, and λ supplies the empty balanced string. Consequently every generated word has equally many a's and b's, and no prefix can contain more b's than a's: for every prefix v, n_a(v)≥n_b(v). This is the one-kind Dyck language described by A.

## Key Points

- S→aSb | SS | λ generates balanced-parenthesis strings with nonnegative prefix balance and final balance zero.

## Why the other options are incorrect

B has the prefix inequality reversed; the word ab generated by aSb already violates B at its first symbol. C and D require unequal total counts, but every production either adds one a and one b or combines already balanced strings.
