# Code of Conduct

## Our standard

This project welcomes learners and contributors of every experience level.
Be respectful, patient and specific. Critique ideas and evidence, not people.
Help others understand corrections instead of using mistakes to embarrass them.

Unacceptable conduct includes harassment, discrimination, threats, deliberate
misinformation, plagiarism, impersonation, spam and promotional content.

## Academic and editorial integrity

Contributors must identify uncertainty, cite the exam session when known and
avoid presenting copied explanations or guessed answers as verified work.
Disagreement about an answer should be resolved with an official key,
independent derivation or a clearly documented dispute.

## Enforcement

Repository maintainers may edit or remove contributions and may restrict
participation when conduct harms the community or the integrity of the guide.
Report problems privately through a maintainer's GitHub contact options when
possible; use GitHub's platform reporting tools for urgent safety concerns.
Maintainers should protect the privacy of people who report incidents.

This code applies in repository issues, pull requests, discussions and other
spaces where someone represents the project.
